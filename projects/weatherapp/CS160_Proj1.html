Project 1 - Weather App                                                                Ashley Park -

Design report

                 

SimplyWeather - Displays weather/forecast for current location, compare between saved locations, receive notifications for hazardous/unexpected weather

Project description

Weather apps offer the user a lot of information, but how much of it does the user actually care about? The goal for my app is to simplify the weather app down to the basic necessities, making it user friendly for the leisure traveler. Taking the targeted audience into consideration, I went for a playful and minimal design. The user can open the app to check the weather at their current location, as well as the forecast to plan accordingly. The user can also save and compare the weather at different locations. The app also offers notifications for hazardous weather.

Interface and interactions

The interface is clean, bright, and simple. Opening the app opens the current location’s weather. The user scrolls up and down the screen and horizontally across the hourly forecast section to see more information. To get to the home page, the user drags diagonally down and can see the current temperature at different locations. Tapping on a location will open up its full weather report. “Add Location” takes the user to a different screen to search and add a new location. “Settings” will take the user to a screen where they can toggle the temperature unit or the notification settings.

Cycle I: Investigation and ideation

MOOD BOARD

Simple - I want my design to feel simple, bright, and clean-cut for the leisure traveler. My moodboard carries out themes such as smooth lines, bright colors, and simple shapes in order to convey these feelings. The smooth lines and simple, flat shapes go hand in hand to create a minimalistic design that should be easy to navigate. I think the use of vector images rather than actual photos gives a more playful vibe that a leisure traveler would enjoy. Similarly, I chose to pair darker colors blue and gray with bright colors pink and yellow to offer playfulness to an otherwise calmer palette. I also wanted to use a lot of white to brighten up the color palette, so I found inspiration in either simple line drawings or colored backgrounds with white drawings.

SUMMARY

In Cycle I I established some basic visual themes I wanted to incorporate throughout my weather app. By doing so I created a “feel” that users would hopefully find easy to use, clean, and visually pleasing. I wanted to stick to a more minimal aesthetic for the leisure travelers, so I put a lot of smooth curves and flat, simple shapes in my moodboard. This translated to how I drew my sketches in the location icons and the use of white space. This part of the process forced me to think about what kind of vibe I wanted my app to give off. It also made me really consider the different choices I had in deciding how I wanted the user to interact with my app, for example in which instances to scroll horizontally vs scroll up and down.

Cycle II: Low-fidelity prototype

Pre-studio wireframe

Post-studio wireframe

Reflection on peer critique - Both peer critiques mentioned that they liked the neat alignment of content, so the overall cleanliness of information is something I want to keep throughout my app. Both also mentioned the lack of a specific functionality dedicated to trip planning, but since I want my app to be weather-centric I decided to forgo those critiques. One critique mentioned that it might be hard for the user to realize they can scroll down to gain more information, so I want to add some sort of icon that can hint that function. Another pointed out that it could be difficult for users to compare weather at different locations, since they’d have to go back and forth between pages. I want to change the way users see their saved location so that it offers more data. Another critique was about the notifications. They liked the idea of two types of notifications but pointed out that there wasn’t much information offered when opening it. Something I have to think about is how to make the notification function more useful.

Reflection on studio critiques

In studio my peers liked the simplistic and playful look I was going for, and most of the dot sticker votes appreciated the wireframe and the idea. For critiques I wanted to get some feedback on the amount of information available on my app. For the leisure traveller, I didn’t want to clutter the interface with information most likely overlooked, but I wasn’t sure if what I had was too little. I got some good suggestions such as including information like humidity, or being able to see more types of comparisons on the home page. Something that was mentioned that I have to think about is the notification. What happens to the notification after the user sees it? Does it go away or stay present somewhere on the app?

SUMMARY - Description of revisions

Post pinterest critique: I added a small icon at the bottom of the screen if there is more information below to help the user infer that a scrolling motion will allow them to see more information. I also added more helpful information to the notifications. The biggest change I made would be the way to add a location and view saved locations. I opted for a home screen rather than side bar so that users can easily compare the temperature at different locations.

Post studio critique: I added humidity and air quality index as part of the weather information, and also added a feature on the home page which would allow users to not only compare the current weather at different locations but also the weekly forecast. I want the user to be able to swipe across the screen and the information in each little location box to swipe from current temperature to weekly forecast. I also added a “Dismiss” button for the notifications so that the notification can disappear after the user acknowledges it.

What changed between wireframe and prototype - The prototype is more simplified than the wireframe. In the full weather report screen I decided to take out the temperature map. In the home page instead of a toggle I opted for a settings button, so that the user can change more things besides the temperature unit. The main difference was implementing the alerts. In the prototype I used a red banner at the top of the screen to notify the alert. By tapping the alert it can take you to a different screen with more information.

DESIGN PRINCIPLES

Color

To portray a light and playful theme I chose this bright yellow along with white and black texts/graphics. I decided to stick to one color throughout the weather report and play around with some of the hues in order to make the design really simple. I think the white and black against the yellow gives it a clean look. The red banner for the alert is really contrasting and attention grabbing, which is important for an alert. The user’s eyes will definitely be drawn to the alert.

Alignment

I aligned a lot of the information vertically on this screen. I wanted to be generous with the spacing and make sure the information wasn’t cluttered. With everything stacked vertically on top of each other, the display of information is simple and clear. I chose to leave the top of the screen more baren, with the current temperature, location name, and location vector image taking up a lot of space so that the user will see the most basic information when opening the screen, but if they want to see more details they can scroll down.
